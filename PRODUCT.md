# Product

## Register

product

## Users

Hunky is for Mac users who work with CD-based game dumps and CHD archives but do not want to install MAME, Homebrew, or run terminal commands. They may be archivists, emulator users, preservation hobbyists, or technically curious collectors. Their context is often batch-oriented: they are checking a folder of disc images, confirming references and integrity, then letting a queue run without babysitting every file.

## Product Purpose

Hunky is a self-contained macOS front end for MAME's `chdman`. It should make CHD creation, extraction, inspection, and verification feel safe, local, and understandable. Success means users can drop files or folders, trust the proposed action, see integrity issues before work starts, and never wonder whether Hunky overwrote or deleted their data.

## Brand Personality

Quiet, precise, trustworthy. Hunky should feel like a native Mac utility with enough domain specificity to show it understands disc archives, but not so much decoration that the work surface feels like a novelty emulator skin.

## Anti-references

Avoid terminal cosplay, neon gaming chrome, sepia archive nostalgia, oversized marketing hero layouts, glassy custom controls, uppercase tracked banners, bordered drop-target rectangles, decorative motion, and color used as ornament. Do not make the app feel like a generic AI-generated dark dashboard or a game launcher.

## Design Principles

- Make safety visible: output location, collision behavior, audit results, and destructive boundaries should be plain.
- Let the queue be the product: the main screen should optimize scanning, comparison, and repeated action.
- Prefer native trust over novelty: standard macOS controls and predictable toolbar behavior beat custom chrome.
- Reserve emphasis for state: use accent, success, caution, and critical colors only when they change what the user should do.
- Keep domain detail close to the work: file type, platform, Redump status, CRC, and output actions belong in rows, not in decorative framing.

## Accessibility & Inclusion

Target at least the native macOS accessibility baseline: keyboard operation, visible focus, VoiceOver labels for icon-only controls, readable contrast in dark mode, and reduced-motion support for all nonessential animation. Do not rely on color alone for Redump, warning, error, or running states.
